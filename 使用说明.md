# GA 使用说明

## 1. 这是什么

GA（桌面应用名为 `A3Agent`）是一个智能体工具，目前同时支持桌面运行和浏览器访问，支持：

- 与大模型聊天和连续执行任务
- 管理多个模型配置并随时切换
- 查看和编辑 SOP 文档
- 管理计划任务

这份文档面向普通使用者，重点介绍“怎么装、怎么配、怎么用”。

---

## 2. 首次使用前你需要准备

至少准备一组可用的大模型接口信息：

- `API Key`
- `API Base`
- `Model`

支持两类常见配置：

- OpenAI 兼容接口
- Claude 接口

---

## 3. 安装与启动

### Windows 系统下安装

1. 双击打开 A3Agent_0.1.0_x64_en-US.msi
2. 选择文件的存放路径（建议不要存放到 C 盘）
3. 首次启动时，如果系统弹出安全提示，请在系统设置中允许打开

### 安装后会发生什么

安装后会在指定文件夹下存放A3-Agent.exe 和 python-backend.exe两个文件，以及卸载执行文件


### 首次启动后会发生什么

双击A3-Agent.exe（或者应用的快捷方式图标），即可启动完整程序，桌面会出现一个悬浮球
![alt text](image.png)

同时，在安装目录下会自动创建用户工作文件夹ga_config，并初始化默认配置。

ga_config通常会包含：

```text
ga_config/
├── mykey.json      # 模型配置
├── memory/         # 记忆与 SOP 相关文件
├── sche_tasks/     # 计划任务
└── temp/           # 临时文件
```

以后你的配置和数据会持续保存在这里,***请确保 ga_config 目录与几个 exe 程序在相同路径下***。

## 4. 首次配置模型

如果没有可用模型，应用通常无法正常完成智能体任务，所以建议先完成这一部分。

### 方式一：在应用界面中配置

进入模型配置界面后，新增或编辑一个模型，填写：

- 类型：`OpenAI 兼容` 或 `Claude`
- `API Base`
- `Model`
- `API Key`

然后按这个顺序操作：

1. 先点“测试”
2. 测试通过后再点“保存”
3. 保存后让应用重新加载模型


### 配置建议

- `API Base` 必须是服务商提供的真实接口地址
- `Model` 必须与该接口实际支持的模型名一致
- `API Key` 填错时，最常见表现是测试失败或发送消息无响应

## 5. 基本使用流程

### 1. 打开应用

启动后，先确认界面中已经加载出可用模型。

### 2. 配置模型

如果没有可用模型，应用通常无法正常完成智能体任务，所以建议先完成这一部分。

点击`模型切换`按钮，进入模型配置界面后，新增或编辑一个模型，填写：

- 类型：`OpenAI 兼容` 或 `Claude`
- `API Base`
- `Model`
- `API Key`

然后按这个顺序操作：

1. 先点“测试”
2. 测试通过后再点“保存”
3. 保存后让应用重新加载模型

*如果测试不通过，则请继续审查配置的正确性，直到通过测试才可以保存并加载模型*

点击`保存并重载`按钮后，会刷新模型列表界面，点击对应模型即可回到主页面开启对话。

### 3. 发送任务

在主聊天输入框中直接输入你的目标，例如：

- 帮我整理这个项目的任务清单
- 根据 SOP 执行一次检查
- 帮我分析当前工作区中的文件

提交后，应用会把请求发送给后端，由智能体持续执行，并通过流式输出返回过程和结果。

### 4. 处理中可执行的操作

任务执行期间，你可以：

- 查看实时输出
- 在需要时手动停止当前任务
- 在模型之间切换

如果界面提示需要人工输入，说明当前步骤需要你补充信息，再继续即可。

## 5. 主要功能说明

### 聊天与任务执行

这是最常用的入口。适合：

- 提问
- 让智能体执行步骤型任务
- 持续观察执行过程

### 模型管理

你可以：

- 新增模型
- 编辑现有模型
- 删除不用的模型
- 在多个模型之间切换

建议至少保留一个稳定可用的模型配置。

### SOP 管理

应用可以读取和编辑 SOP 文档。

适合场景：

- 把常用操作沉淀成 SOP
- 让智能体按既定步骤执行
- 持续维护团队内部流程

### 计划任务

应用内可以查看和维护计划任务文件，常见状态包括：

- `pending`
- `running`
- `done`

你可以新建、编辑、删除计划任务，并控制调度器开关。

### 待办事项

应用支持简单的待办内容查看和保存，适合记录临时任务或运行备注。

## 6. 常见使用建议

### 提示词尽量具体

比起“帮我处理一下”，更推荐：

- 目标是什么
- 输入在哪里
- 希望输出成什么样
- 有哪些限制条件

### 一个任务只做一件大事

复杂任务建议拆分，效果通常更稳定。例如：

1. 先让它阅读材料
2. 再让它输出结论
3. 最后让它整理成固定格式

### 重要数据请定期备份

建议定期在其他路径备份ga_config文件夹：

这样可以一并保留：

- 模型配置
- 计划任务
- SOP
- 记忆数据
