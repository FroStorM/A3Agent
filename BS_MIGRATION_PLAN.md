# GA BS 化工作清单

目标：把当前以 `Tauri/PyQt + 本地 Python` 为主的桌面运行方式，调整为可直接部署到服务器、通过浏览器访问的 `BS` 架构。

## 第一阶段：最小可用 BS 版本

- [x] 梳理当前架构，确认真正的业务核心仍是 `FastAPI + Web 前端`
- [x] 明确桌面壳与业务层的边界
- [x] 让 `FastAPI` 直接托管前端静态页面
- [x] 让前端同时兼容“桌面注入口径”和“浏览器直连口径”
- [x] 增加面向服务器部署的启动入口
- [x] 补充最基础的 Web 部署文档

## 第二阶段：部署与配置整理

- [ ] 统一运行时目录、工作区目录和 `ga_config` 目录的约定
- [ ] 梳理哪些配置必须保存在服务器本地，哪些可以通过环境变量注入
- [ ] 为反向代理场景补充 Host、Port、跨域和静态资源说明
- [ ] 增加生产部署建议（如 `systemd`、Nginx、Docker 预留）

## 第三阶段：去桌面壳耦合

- [ ] 评估 `Tauri` 是否保留为兼容入口，还是下线为历史模式
- [ ] 评估 `PyQt` 启动器、悬浮球等桌面特性是否保留
- [ ] 清理只服务桌面打包的脚本和说明，避免后续维护混乱

## 第四阶段：稳定性与上线准备

- [ ] 补充基础启动验证脚本
- [ ] 验证 SSE 流式输出在浏览器和代理层下的兼容性
- [ ] 验证文件读写、SOP、计划任务在服务器部署目录下的权限
- [ ] 整理上线检查清单
